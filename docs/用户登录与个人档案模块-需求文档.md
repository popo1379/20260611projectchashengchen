# 用户登录与个人档案模块 — 需求文档

---

## 一、项目背景与目标

### 1.1 现状问题

当前本命星盘模块的用户使用路径：

```
首页 → 录入出生日期/时间/城市 → 查看星盘 → 查看运势 → 退出小程序
再次使用 → 重新录入所有信息
```

**痛点分析：**

| 问题 | 影响 |
|------|------|
| 每次打开都需要重新录入出生信息 | 用户流失率升高，尤其对于"每日查看运势"场景 |
| 无法保存历史查询记录 | 用户无法回顾过往的运势解读 |
| 无法建立用户画像 | 无法做个性化推荐、命理档案长期追踪 |
| 无账号体系 | 无法实现多设备同步、云端备份 |

### 1.2 目标

在现有星盘功能基础上，新增用户登录与个人档案模块，实现：

- **一次录入，长期使用**：用户录入一次出生信息，后续打开直接看到个人星盘
- **每日运势直达**：用户打开小程序首页，直接展示当日运势，无需跳转
- **历史记录追溯**：保存用户每次查询的日期和运势快照
- **轻量级账号体系**：不强制注册，但提供微信一键登录降低门槛

## 二、用户场景与故事

### 场景一：快速查看当日运势

> 小王习惯每天早上看一眼自己的运势。他打开小程序，首页直接显示"小王，今日运势"，他不需要做任何操作就知道今天的能量走向。

**流程：**
1. 用户首次打开小程序 → 微信一键登录 → 录入出生信息
2. 此后每次打开小程序 → 自动识别用户 → 首页展示个人当日运势

### 场景二：查看完整星盘档案

> 小李想跟朋友分享自己的星盘解读。他打开小程序，点击"我的星盘"，看到完整的太阳/月亮/上升三要素，以及逐行星的解读。

**流程：**
1. 用户进入"我的星盘"页面 → 查看完整本命盘信息
2. 可将星盘截图或分享链接发给朋友

### 场景三：回顾历史运势

> 小张想看看上周自己的运势如何。他进入"历史记录"，选择上周任意一天，查看当时的运势快照。

**流程：**
1. 用户进入"历史记录"页面 → 日历/列表形式展示历史记录
2. 点击任意日期 → 查看当时的运势快照

### 场景四：新用户引导录入

> 首次使用小程序的阿华，没有任何账号。他点击首页的"查命理"入口，系统引导他通过微信登录并录入出生信息。

**流程：**
1. 首页点击"查命理" → 判断是否已登录
2. 未登录 → 微信一键登录 → 录入信息页
3. 已登录但无出生信息 → 直接进入录入页
4. 已登录且有信息 → 跳转至星盘结果页

## 三、功能需求详述

### 3.1 账号与登录

#### F1：微信一键登录

- 基于微信小程序 `wx.login` + 微信用户授权
- 不需要独立注册流程，点击即完成登录
- 获取用户微信头像、昵称作为默认展示信息
- **可拒绝登录**：用户可选择跳过，仅使用基础功能（但无法保存个人档案）

#### F2：登录状态持久化

- 登录态通过微信自身机制或本地存储 token 保持
- 切换微信账号时，小程序自动识别并同步用户数据
- 退出登录后，本地缓存的出生信息保留（用户可选择清除）

#### F3：多账号区分

- 同一设备登录不同微信账号 → 数据完全隔离
- 支持一个微信账号为主账号，记录多个出生档案（如帮家人查询）

### 3.2 个人档案管理

#### F4：出生信息录入

需要录入的字段：

| 字段 | 是否必填 | 说明 | 备注 |
|------|---------|------|------|
| 出生日期 | 必填 | 年 / 月 / 日 | 使用系统日期选择器 |
| 出生时间 | 选填 | 时 / 分 | 不填写时标注"时间未知，上升星座仅供参考" |
| 出生城市 | 必填 | 城市名称 + 经纬度 | 支持模糊搜索，当前已有全国地级市数据库 |

录入后可随时修改。

#### F5：档案管理（一人多档）

- 一个微信账号可保存 **最多 3 份**出生档案
- 每份档案独立命名（如"我的星盘"、"妈妈的命盘"）
- 切换档案后，星盘和运势自动切换
- 可设置默认档案，每次打开自动加载

#### F6：档案导出与分享

- 支持生成分享卡片（图片形式，包含星盘摘要信息）
- 可复制档案链接（跳转回小程序查看）

### 3.3 首页改版

#### F7：登录后首页直接展示运势

**改版前：**
```
首页 → 查命理入口 → 录入页 → 结果页 → 运势
```

**改版后（已登录用户）：**
```
打开小程序 → 首页直接展示：
┌─────────────────────────────┐
│  早安，阿华                  │
│  今日运势：★★★☆☆（不错）    │
│  ──────────────────────     │
│  今天的水星与你的太阳形成     │
│  三分相...                   │
│  [查看完整运势]              │
└─────────────────────────────┘
```

- 首页展示用户昵称、今日运势摘要（1-2 句话）
- 点击可跳转至完整星盘结果页
- 未登录用户保留现有入口结构

#### F8：首页快捷入口

| 入口 | 描述 |
|------|------|
| 今日运势 | 快速查看当天运势（已登录用户默认展示） |
| 我的星盘 | 完整本命盘信息 |
| 历史记录 | 查看过往运势快照 |
| 档案管理 | 添加/切换/删除档案 |

### 3.4 历史记录

#### F9：运势快照存储

- 每次用户查看某一天的运势时，自动保存一条快照记录
- 快照内容包括：
  - 日期
  - 运势评分和星级
  - 运势摘要文本
  - 当时的主要行运相位事件
- 支持按月份筛选查看

#### F10：历史运势对比

- 选择两个日期，横向对比两天的运势变化
- 展示差异（如：从★★★升到★★★★，主要变化在木星相位）

### 4.4 数据存储策略

| 数据类型 | 存储位置 | 说明 |
|---------|---------|------|
| 用户基础信息（openid、昵称） | 微信服务器 | 通过微信登录获取，不可伪造 |
| 出生档案（加密存储） | 小程序云开发数据库 / 自行部署后端 | 需 HTTPS 传输 |
| 运势快照 | 小程序云开发数据库 / 自行部署后端 | 每日生成，自动入库 |
| 本地缓存（快速读取） | 微信本地存储 wx.setStorageSync | 用户拒绝登录时的降级方案 |

## 五、技术方案评估

### 方案 A：微信云开发（推荐）

**优点：**
- 无需自建服务器，成本低
- 云数据库直接支持用户 openid 索引，数据隔离天然安全
- 云函数处理敏感逻辑（如出生信息加密存储）
- 控制台提供免费额度，初期足够

**缺点：**
- 依赖腾讯云，迁移成本高
- 免费额度有限，用户量增长后有成本

**适合阶段：** MVP（最小可行产品）、用户量 < 10 万

---

### 方案 B：自建后端 + 数据库

**优点：**
- 完全自主可控
- 可扩展性强，支持复杂业务
- 可对接 Web/App 多端

**缺点：**
- 需要服务器和运维成本
- 开发周期长
- 需要处理用户认证、接口安全等

**适合阶段：** 用户量 > 10 万，或有多端需求

---

### 方案 C：纯本地存储（降级方案）

**优点：**
- 开发最快
- 无服务器成本
- 隐私性最好（数据不出设备）

**缺点：**
- 换手机/换设备数据丢失
- 无法多设备同步
- 无法做历史记录云端回溯

**适合阶段：** 隐私敏感用户，或作为"拒绝登录"用户的降级体验

## 六、隐私与安全

### 6.1 出生信息敏感性

出生日期 + 出生时间 + 出生地点组合起来可精确到分钟级别的时空坐标，具有较高隐私敏感性。

**处理措施：**

| 措施 | 说明 |
|------|------|
| 数据加密存储 | 出生时间等敏感字段在云端加密存储 |
| 最小化获取 | 仅获取微信昵称和头像不过度授权 |
| 用户删除权 | 用户可随时删除自己的全部数据 |
| 隐私政策公示 | 在登录/录入页面展示隐私政策链接 |

### 6.2 微信登录合规

- 遵守《微信小程序用户服务协议》和《微信个人信息保护指引》
- 不向第三方透露用户 openid
- 不将出生信息用于广告推送或用户画像商业化

## 九、待确认事项

在开发前需要你确认以下决策点：

| 问题 | 选项 |
|------|------|
| **数据存储方案** | A. 微信云开发（推荐） / B. 自建后端 / C. 本地优先+云端可选 |
| **账号强制程度** | A. 不强制登录（拒绝登录也能用，但数据不保存）/ B. 强制登录才能使用核心功能 |
| **多档案限制** | A. 免费用户最多 1 档 / B. 免费用户最多 3 档 |
